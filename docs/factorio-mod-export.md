# Factorio Mod Export

The editor can download the current GUI model as a local Factorio preview mod.
This is a parity-testing loop, not a production mod package.

## What It Exports

Use the `Download mod` button in the Lua Output panel after creating a Window.
The browser downloads:

```text
labtorio_gui_preview_0.1.0.zip
`- labtorio_gui_preview_0.1.0/
   |- info.json
   |- control.lua
   `- gui.lua
```

`gui.lua` is generated by the same `renderWindowLua(model)` projection shown in
the Lua Output panel. The wrapper files only make that Lua loadable as a local
Factorio mod.

The generated mod uses Factorio `2.0` metadata and depends only on `base`. It
does not include Factorio images, Wube CSS, bundled examples, or any generated
runtime artifacts in this repository.

## Manual Test Loop

1. Build the layout in the web editor.
2. Keep Lua Output enabled and download the mod zip.
3. Copy the zip into the Factorio mods directory for the local install.
4. Restart or reload Factorio so the mod is discovered.
5. Enable `Labtorio GUI Preview`, then load or create a save.
6. Compare the in-game GUI against the browser preview and any reference
   screenshots or style-inspector evidence.

Useful graphical checks remain Factorio-side: `Ctrl+F6` for the GUI style
inspector, `Ctrl+F5` for bounding boxes, and `Ctrl+F7` for shadows.

## Limits

This export only proves that the current shared model can be wrapped quickly for
in-game inspection. It does not automate Factorio launch, screenshot capture,
pixel comparison, or visual acceptance. Those remain future parity-harness work.

If a browser/in-game difference appears, treat the generated mod as evidence for
the model-to-Lua path and inspect whether the mismatch comes from browser
renderer approximation, missing model fields, unsupported atom behavior, or
Factorio layout/style rules not yet captured.
