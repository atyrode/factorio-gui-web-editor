# Factorio Mod Export

The editor can download the current GUI model as a local Factorio preview mod.
This is a parity-testing loop, not a production mod package.

For reversible editor work, use the export drawer's `Download design` and
`Import` controls. Raw `*.labtorio-gui.json` files preserve the tool-authored
structured model, and compatible preview/tool-authored zip packages can also be
imported when they contain `design.labtorio-gui.json`. Generated Lua remains a
projection from that model and is not parsed back into the editor.

## What It Exports

Use the `Download mod` button in the Lua Output panel after creating a Window.
The browser downloads:

```text
labtorio_gui_preview_0.1.0.zip
`- labtorio_gui_preview_0.1.0/
   |- info.json
   |- control.lua
   |- gui.lua
   |- labtorio-gui-package.json
   `- design.labtorio-gui.json
```

`gui.lua` is generated by the same `renderWindowLua(model)` projection shown in
the Lua Output panel. `design.labtorio-gui.json` is the same structured design
payload used by the export drawer, included so a preview zip can carry the
tool-authored source model and be imported back into the editor.
`labtorio-gui-package.json` declares the package schema, file ownership,
design-file entry, style-catalog provenance, and reserved hook metadata. The
wrapper files only make the Lua loadable as a local Factorio mod.

## Package Boundary

Current preview packages use `labtorio-gui-package.v0`:

```text
labtorio-gui-package.json:
  tool-owned package manifest and provenance
design.labtorio-gui.json:
  tool-owned canonical editable GUI design
gui.lua:
  generated Lua projection; do not edit as the canonical source
control.lua:
  tool-owned preview runtime wrapper
info.json:
  tool-owned preview mod metadata
```

The manifest also includes a reserved `labtorio-gui-hooks.v0` section with
empty `actions` and `events` arrays. That section is a forward-compatible slot
for behavior hook names and event/action metadata; the current preview export
does not implement behavior wiring.

Manual mod code should live outside the tool-owned preview files, or in a later
documented user-owned package location. Editing `gui.lua` can still be useful
for experiments, but those changes are generated-code edits and are not
imported back into the editor. The editable source of truth remains the design
payload referenced by the package manifest.

The generated mod uses Factorio `2.0` metadata and depends only on `base`. It
does not include Factorio images, Wube CSS, bundled examples, or any generated
runtime artifacts in this repository.

## Manual Test Loop

1. Build the layout in the web editor.
2. Keep Lua Output enabled and download the mod zip.
3. Copy the zip into the Factorio mods directory for the local install.
4. Restart or reload Factorio so the mod is discovered.
5. Enable `Labtorio GUI Preview`, then load or create a save.
6. Compare the in-game GUI against the browser preview and any reference
   screenshots or style-inspector evidence.

Useful graphical checks remain Factorio-side: `Ctrl+F6` for the GUI style
inspector, `Ctrl+F5` for bounding boxes, and `Ctrl+F7` for shadows.

## Limits

This export only proves that the current shared model can be wrapped quickly for
in-game inspection. The embedded design file is inert as far as Factorio is
concerned; it is not read by `control.lua`, and the editor imports the package
manifest/design JSON rather than parsing generated Lua. Design-only zips from
older exports can still be imported with a warning, but manifest-backed
packages are the supported boundary. This export does not automate
Factorio launch, screenshot capture, pixel comparison, or visual acceptance.
Those remain future parity-harness work.

If a browser/in-game difference appears, treat the generated mod as evidence for
the model-to-Lua path and inspect whether the mismatch comes from browser
renderer approximation, missing model fields, unsupported atom behavior, or
Factorio layout/style rules not yet captured.
